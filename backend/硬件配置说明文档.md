# 硬件设备配置指南

## 1. 硬件配置文件位置

配置文件位于：`backend/src/main/resources/application.yml`

所有硬件相关的配置参数都在 `hardware` 节点下。

## 2. MQTT Broker 配置

### 2.1 MQTT 连接参数

```yaml
mqtt:
  host: broker.emqx.io        # MQTT Broker地址
  port: 1883                  # MQTT端口
  username: ""                # 用户名（根据实际硬件设置）
  password: ""                # 密码（根据实际硬件设置）
  client-id: qiananzhikui-test001  # 客户端ID
  topic-subscribe: device/+/data,device/+/alarm,device/+/sos  # 订阅主题
  timeout: 60                 # 连接超时（秒）
  keepalive: 60               # 心跳间隔（秒）
```

### 2.2 配置说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| host | String | 是 | MQTT Broker服务器地址 | `broker.emqx.io` 或 `192.168.1.100` |
| port | Integer | 是 | MQTT服务端口 | `1883` (非加密) 或 `8883` (加密) |
| username | String | 否 | 认证用户名 | `device_user` |
| password | String | 否 | 认证密码 | `secure_password` |
| client-id | String | 是 | 客户端唯一标识 | `helmet-server-001` |
| topic-subscribe | String | 是 | 订阅的主题（支持通配符） | `device/+/data` |
| timeout | Integer | 否 | 连接超时时间，默认60秒 | `60` |
| keepalive | Integer | 否 | 心跳间隔，默认60秒 | `60` |

## 3. 硬件数据配置

### 3.1 数据验证参数

```yaml
hardware:
  data:
    heartbeat-timeout: 30000  # 设备心跳超时（毫秒）
    data-validate: true      # 是否启用数据校验
    max-heartrate: 250        # 心率最大值
    min-heartrate: 30         # 心率最小值
    max-temperature: 45.0     # 体温最大值
    min-temperature: 30.0     # 体温最小值
```

### 3.2 配置说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| heartbeat-timeout | Long | 否 | 设备心跳超时时间，默认30000ms | `30000` |
| data-validate | Boolean | 否 | 是否启用数据校验，默认true | `true` |
| max-heartrate | Integer | 否 | 心率数据最大值 | `250` |
| min-heartrate | Integer | 否 | 心率数据最小值 | `30` |
| max-temperature | Double | 否 | 体温数据最大值 | `45.0` |
| min-temperature | Double | 否 | 体温数据最小值 | `30.0` |

## 4. 设备通信配置

### 4.1 命令发送参数

```yaml
hardware:
  communication:
    command-retry-count: 3      # 命令重试次数
    command-retry-delay: 1000  # 重试间隔（毫秒）
    command-timeout: 5000      # 命令响应超时（毫秒）
    max-message-size: 1024     # 消息最大字节数
```

### 4.2 配置说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| command-retry-count | Integer | 否 | 命令发送失败重试次数，默认3 | `3` |
| command-retry-delay | Long | 否 | 每次重试的间隔时间，默认1000ms | `1000` |
| command-timeout | Long | 否 | 命令响应超时时间，默认5000ms | `5000` |
| max-message-size | Integer | 否 | 单条消息最大字节数 | `1024` |

## 5. 设备型号配置

### 5.1 设备型号参数

```yaml
hardware:
  device-models:
    helmet-v1:
      manufacturer: "头盔厂商A"
      firmware-version: "1.0.0"
      support-features:
        - led_control
        - voice_broadcast
        - sos_alarm
```

### 5.2 支持的功能列表

| 功能标识 | 功能名称 | 说明 |
|---------|---------|------|
| led_control | LED控制 | 控制头盔LED灯的开关和闪烁 |
| voice_broadcast | 语音播报 | 向头盔发送语音播报消息 |
| sos_alarm | SOS报警 | 触发或取消SOS紧急报警 |
| heartrate_monitor | 心率监测 | 接收和记录心率数据 |
| temperature_monitor | 体温监测 | 接收和记录体温数据 |
| fall_detection | 摔倒检测 | 接收摔倒检测报警 |
| gps_tracking | GPS定位 | 接收GPS定位数据 |

## 6. 硬件设备数据格式要求

### 6.1 设备上报数据格式（MQTT主题：device/{deviceSn}/data）

```json
{
  "deviceSn": "DQ2025001",
  "riderId": 1,
  "heartRate": 75,
  "temperature": 36.5,
  "posture": "normal",
  "speed": 15.5,
  "longitude": "103.8232",
  "latitude": "30.6745",
  "fallStatus": 0
}
```

### 6.2 字段说明

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| deviceSn | String | 是 | 设备序列号 | `DQ2025001` |
| riderId | Long | 是 | 骑手ID | `1` |
| heartRate | Integer | 否 | 心率值（bpm） | `75` |
| temperature | Decimal | 否 | 体温值（℃） | `36.5` |
| posture | String | 否 | 姿态状态 | `normal` / `fall` |
| speed | Decimal | 否 | 骑行速度（km/h） | `15.5` |
| longitude | String | 否 | 经度 | `103.8232` |
| latitude | String | 否 | 纬度 | `30.6745` |
| fallStatus | Integer | 否 | 摔倒状态（0:正常, 1:摔倒） | `0` |

### 6.3 心跳数据格式（MQTT主题：device/{deviceSn}/heartbeat）

```json
{
  "deviceSn": "DQ2025001",
  "timestamp": 1700000000000,
  "battery": 85,
  "signalStrength": -65
}
```

### 6.4 报警数据格式（MQTT主题：device/{deviceSn}/alarm）

```json
{
  "deviceSn": "DQ2025001",
  "riderId": 1,
  "alarmType": "temperature",
  "alarmContent": "体温异常",
  "longitude": "103.8232",
  "latitude": "30.6745"
}
```

### 6.5 SOS数据格式（MQTT主题：device/{deviceSn}/sos）

```json
{
  "deviceSn": "DQ2025001",
  "riderId": 1,
  "sosStatus": true,
  "longitude": "103.8232",
  "latitude": "30.6745",
  "timestamp": 1700000000000
}
```

## 7. 命令下发格式

### 7.1 LED控制命令（MQTT主题：device/{deviceSn}/cmd/led）

```json
{
  "deviceSn": "DQ2025001",
  "commandType": "led_control",
  "requestId": "1700000000000_1234",
  "status": 1,
  "timestamp": 1700000000000
}
```

- `status`: 0=关闭, 1=开启, 2=闪烁

### 7.2 语音播报命令（MQTT主题：device/{deviceSn}/cmd/voice）

```json
{
  "deviceSn": "DQ2025001",
  "commandType": "voice_broadcast",
  "requestId": "1700000000000_1234",
  "message": "前方有障碍物，请注意安全",
  "timestamp": 1700000000000
}
```

- `message`: 播报内容（最多200字符）

### 7.3 音量调节命令（MQTT主题：device/{deviceSn}/cmd/volume）

```json
{
  "deviceSn": "DQ2025001",
  "commandType": "voice_volume",
  "requestId": "1700000000000_1234",
  "volume": 80,
  "timestamp": 1700000000000
}
```

- `volume`: 音量值（0-100）

### 7.4 SOS控制命令（MQTT主题：device/{deviceSn}/cmd/sos）

```json
{
  "deviceSn": "DQ2025001",
  "commandType": "sos_alarm",
  "requestId": "1700000000000_1234",
  "enabled": true,
  "timestamp": 1700000000000
}
```

- `enabled`: true=启用SOS, false=取消SOS



## 8. 硬件连接测试方法

### 8.1 测试前准备

1. **确认MQTT Broker运行正常**
   ```bash
   # 使用MQTT客户端工具连接
   mosquitto_pub -h broker.emqx.io -p 1883 -t test -m "hello"
   ```

2. **检查网络连通性**
   ```bash
   # 测试MQTT端口是否开放
   telnet broker.emqx.io 1883
   ```

3. **准备测试设备**
   - 设备序列号（Device SN）：例如 `DQ2025001`
   - 设备已连接到MQTT Broker
   - 设备已订阅命令主题

### 8.2 数据接收测试

#### 8.2.1 模拟设备上报数据

使用MQTT客户端工具模拟设备发送数据：

```bash

  -t "device/DQ2025001/data" \
  -m '{
    "deviceSn": "DQ2025001",
    "riderId": 1,
    "heartRate": 75,
    "temperature": 36.5,
    "posture": "normal",
    "speed": 15.5,
    "longitude": "103.8232",
    "latitude": "30.6745",
    "fallStatus": 0
  }'

# 发送心跳
mosquitto_pub -h broker.emqx.io -p 1883 \
  -t "device/DQ2025001/heartbeat" \
  -m '{
    "deviceSn": "DQ2025001",
    "timestamp": 1700000000000,
    "battery": 85
```

## 9. 常见问题排查

### 9.1 MQTT连接失败

**症状**：应用启动时显示 "MQTT connection failed"

**排查步骤**：
1. 检查MQTT Broker是否运行：`telnet broker.emqx.io 1883`
2. 检查防火墙是否开放1883端口
3. 检查host配置是否正确
4. 检查网络是否可访问

**解决方案**：
- 如果使用本地MQTT Broker，修改 `mqtt.host` 为本地地址
- 如果需要认证，配置 `mqtt.username` 和 `mqtt.password`

### 9.2 设备数据接收不到

**症状**：设备发送数据，但应用日志没有收到消息

**排查步骤**：
1. 确认MQTT订阅主题是否正确：`device/+/data`
2. 确认设备发送的主题格式：`device/{deviceSn}/data`
3. 使用MQTT客户端订阅同一主题验证

**解决方案**：
- 检查设备端MQTT客户端是否成功订阅了数据主题
- 确认设备发送的JSON格式是否符合规范

### 9.3 命令发送失败

**症状**：调用API返回 "命令发送失败"

**排查步骤**：
1. 检查MQTT连接状态
2. 检查命令主题是否正确：`device/{deviceSn}/cmd/{type}`
3. 查看应用日志中的错误信息

**解决方案**：
- 确保MQTT客户端处于连接状态
- 检查设备是否在线（心跳超时）
- 增加重试次数和超时时间

### 9.4 数据校验失败

**症状**：应用日志显示 "Invalid device data format"

**排查步骤**：
1. 检查设备序列号是否为空或格式错误
2. 检查数据值是否在合理范围内
3. 查看完整的JSON数据

**解决方案**：
- 确保设备序列号长度为5-32个字符
- 确保心率值在30-250之间
- 确保体温值在30-45℃之间
- 可以在配置中关闭数据校验：`hardware.data.data-validate: false`
